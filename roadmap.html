NHID-Clinical v1.4: Cryptographic Agent Identity

v1.3 establishes disclosure and audit. v1.4 adds the missing piece: a way to cryptographically verify that an AI agent is actually authorized by the provider it claims to represent.

Status: Reference implementation on main — 25 tests passing, all identity features complete. Full spec integration and production API in progress. Available for early testing now.

Where v1.3 Ends and v1.4 Begins

Capability v1.3 (current) v1.4 (preview)

Provider authorization ❌ Not verifiable ✅ Ed25519 delegation chain
Revocation ❌ Not supported ✅ HTTP revocation list

What's in v1.4

Agent Passport System (APS)

Ed25519-based cryptographic identity for AI agents. Each agent has a key pair. Providers sign a delegation granting the agent specific scopes (e.g., eligibility, claim_status). Based on IETF draft standards.

Delegation Chains with Scope Attenuation

A provider authorizes an agent with limited permissions. The agent cannot exceed its granted scope. Delegations have explicit TTLs — a 24-hour delegation cannot be replayed the next day.

Revocation Lists

Compromised agents can be revoked immediately. Payers check a simple HTTP endpoint before accepting a passport. No central registry required in v1.4 — each provider hosts their own revocation list.

Call Integration

Passport tokens delivered via DTMF or SIP headers — no change to the voice interaction itself. The payer's system validates the token out-of-band before the call proceeds to data exchange.

Try It Now

The identity module is on main. 25 tests covering the full feature set: NPI validation, delegation chains, scope enforcement, per-delegation revocation, call-bound nonces, and 1000-verification performance.

git clone https://github.com/NHID-Clinical/NHID-Clinical.git
cd NHID-Clinical
pip install -r requirements.txt
python -m pytest tests/test_identity.py -v
