Pilot Guide · 90 Days · Shadow Mode

For Payers: 90-Day Shadow Pilot

A practical starting point for payer operations teams who want to evaluate AI voice agent transparency with one vendor before going further.

Important: No payer has piloted or adopted NHID-Clinical yet. You would be among the first. This is a voluntary reference model from a solo founder.

What You Get

Metric Today (typical) Target with NHID-Clinical
Verification latency ("are you human?") 3–5 minutes or call terminated <5 seconds
Audit effort per vendor Manual call review (hours) ~2 minutes (run test suite)
RFP disclosure language Custom per vendor One standard clause

90-Day Shadow Pilot Structure

Weeks 1–2 · RFP / Vendor Ask

Add this clause to your next voice AI vendor RFP or BAA amendment:

"The vendor's AI agent SHALL produce NHID-Clinical v1.3 JSON trace logs for all B2B calls, including disclosure timestamps and opt-out handling. The payer may run the open-source conformance test suite against vendor output."

Weeks 3–6 · Sandbox Testing

Ask the vendor to run the conformance test suite in their sandbox and send the resulting JSON traces (100 calls recommended).

Weeks 7–10 · Validation

Run the test suite yourself against the vendor's logs. Check for:

  • Disclosure before any operational data is requested
  • No deceptive artifacts
  • Opt-out / human escalation works

Weeks 11–12 · Measure Results

Compare before vs after:

  • Average verification latency
  • Escalation call volume due to identity uncertainty

What This Pilot Is NOT For

  • Live decision-making or prior auth automation
  • Replacing call center agents
  • Writing data back into your core systems
  • Agent identity verification / spoofing prevention
This is shadow-mode only. It logs and validates — it does not decide.
