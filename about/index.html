About NHID-Clinical

Built from the moment the script stopped working.

NHID-Clinical exists because healthcare administrative voice workflows now receive AI callers that sound human, act at scale, and often disclose too late.

Built by Brianna Baynard

Created from direct experience answering AI voice agent calls in payer-provider administrative workflows.

The problem is the workflow, not the AI.

Healthcare should not simply ban automation from administrative work. Eligibility checks, claim status calls, prior authorization follow-up, and billing tasks are repetitive workflows where automation can reduce burden. The issue is that AI voice agents have entered a trust-based channel without a basic trust model.

NHID-Clinical defines the minimum behavioral baseline for that channel. If the caller is automated, it should say so before asking for data. If it represents a provider, it should identify the organization and purpose of the call. If the call cannot continue, it should hand off to a human cleanly with a reference ID.

Why the scope is narrow

This is not a full AI law, a HIPAA replacement, or a general identity platform. It addresses one concrete moment: an automated voice agent calls a payer, says it represents a provider, and asks for administrative information.
